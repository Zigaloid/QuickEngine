## Slope Validation Flow

```
Is character grounded?
    ?? NO  ? Apply desired velocity unchanged
    ?? YES ? Check slope
             ?? Get ground normal horizontal component
             ?? Check: moving into surface?
             ?   ?? NO  ? Allow full movement
             ?   ?? YES ? Compare slope angles
             ?            ?? Actual <= Max?
             ?            ?  ?? YES ? Allow movement
             ?            ?  ?? NO  ? Project parallel
             ?            ?? Return adjusted movement
             ?? Apply blended adjusted velocity
```

## Data Flow: Single Frame Example

### Setup
- Player pressing forward and right
- Moving speed: 5 m/s
- Velocity blend: 0.25
- Current Y velocity: 0.2 m/s (just fell, approaching ground)
- Max slope angle: 45°

### Step 1: Ground State
```
Y velocity (0.2) <= threshold (0.5) ? m_isGrounded = true ?
```

### Step 2: Movement Allowed
```
canMove = m_controlMovementDuringJump || m_isGrounded
canMove = true || true = true ?
```

### Step 3: Desired Velocity
```
Input X: 0.7, Input Z: -0.7 (normalized diagonal)
Desired velocity:
  moveX = 0.7 * 5.0 = 3.5
  moveZ = -0.7 * 5.0 = -3.5

Slope check (not steep) ? accepted as-is
desiredVel = (3.5, 0, -3.5)
```

### Step 4: Velocity Blending
```
Current velocity: (0, 0.2, 0)

newVel.x = (1 - 0.25) * 0 + 0.25 * 3.5
         = 0.75 * 0 + 0.25 * 3.5
         = 0.875

newVel.y = 0.2 (unchanged)

newVel.z = (1 - 0.25) * 0 + 0.25 * (-3.5)
         = 0.875 * (-3.5) / 3.5  (simplified)
         = -0.875

Result: (0.875, 0.2, -0.875)
```

### Step 5: Jump (if queued)
```
Jump not queued ? skip
```

### Step 6: Apply Physics
```
SetLinearVelocity(bodyId, (0.875, 0.2, -0.875))
```

### Step 7: Physics Update
```
Jolt applies gravity: (0.875, 0.2 - 9.81*deltaTime, -0.875)
If on ground and steep slope: might stop at contact
Character moves with blended velocity
Next frame reads new position
```

## Parameter Tuning Decision Tree

```
Does movement feel twitchy?
?? YES ? Decrease blend factor (0.25 ? 0.15)
?? NO  ? Continue

Does character feel sluggish?
?? YES ? Increase blend factor (0.25 ? 0.35) or speed
?? NO  ? Continue

Can character climb unintended slopes?
?? YES ? Decrease max slope angle (45 ? 35)
?? NO  ? Continue

Can steer too much mid-air?
?? YES ? SetControlMovementDuringJump(false)
?? NO  ? Continue

Does jump feel weak?
?? YES ? Increase jump impulse
?? NO  ? Movement setup complete! ?
```

## Comparison: Old vs New Implementation

### Old Velocity Application
```cpp
void OnUpdate() {
    float moveX = m_inputX * m_moveSpeed;      // Instant desired
    float moveZ = -m_inputZ * m_moveSpeed;

    JPH::Vec3 vel = bi.GetLinearVelocity(bodyId);
    vel = JPH::Vec3(moveX, vel.GetY(), moveZ);  // Direct replace
    bi.SetLinearVelocity(bodyId, vel);
}
```

**Problems:**
- Velocity changes instantly
- No acceleration phase
- Twitchy to steer
- No slope awareness
- Unreliable ground detection

### New Velocity Application
```cpp
void OnUpdate() {
    UpdateGroundState();                           // Evaluate support
    bool canMove = m_controlMovementDuringJump || m_isGrounded;

    JPH::Vec3 desired = GetDesiredVelocity();      // With slope check
    JPH::Vec3 current = bi.GetLinearVelocity(bodyId);

    JPH::Vec3 newVel(
        (1-blend)*current.x + blend*desired.x,
        current.y,
        (1-blend)*current.z + blend*desired.z
    );                                             // Smooth blending

    bi.SetLinearVelocity(bodyId, newVel);
}
```

**Benefits:**
- Smooth acceleration
- More game-feel
- Responsive but not twitchy
- Slope validation
- Proper ground check

## Thread Safety

- **Input thread**: InputActionManager (separate)
- **Simulation thread**: PhysicsManager (may be concurrent)
- **Game thread**: This component runs in main thread during OnUpdate()

No thread synchronization issues since:
- Velocity reads are atomic
- Velocity sets are atomic
- No shared state with other components
- Physics system handles synchronization

## Conclusion

The new character controller provides:
1. **Better game feel** through velocity blending
2. **Slope awareness** preventing unintended climbing
3. **Flexible control** with configurable air movement
4. **Clean architecture** with separated concerns
5. **Easy tuning** with well-documented parameters
