1. 🌟 Overview

The Reflection System provides automatic serialization and deserialization capabilities for C++ objects. It supports multiple file formats (JSON, Binary) and integrates seamlessly with the Component System for game engine development.

Key Features:
  • ✅ Automatic serialization/deserialization
  • ✅ Multiple file format support (JSON, Binary)
  • ✅ Type-safe error handling with Result<T>
  • ✅ Component system integration
  • ✅ Comprehensive error reporting
  • ✅ Memory-safe smart pointer support

2. 🏗️ Core Classes

CReflectedBase

The base class for all reflectable objects. All classes that need serialization must inherit from this class.

class CReflectedBase
// Core serialization methods bool Read(const char* fileName); bool Write(const char* fileName); // Safe methods with Result<T> return type Reflection::Result<bool> SafeRead(const std::string& fileName); Reflection::Result<bool> SafeWrite(const std::string& fileName); // Virtual methods for inheritance virtual void ReadMembers(IRFL_Parser& doc); virtual void WriteMembers(IRFL_Parser& doc); virtual const char* GetClassName() const;

IRFL_Parser

Abstract interface for different serialization formats. Implementations include JSON and Binary parsers.

Result<T>

Type-safe error handling wrapper that either contains a value or an error.

auto result = obj.SafeRead("file.json"); if (result.IsSuccess()) { bool success = result.GetValue(); } else { const auto& error = result.GetError(); std::cout << "Error: " << error.message << std::endl; }

5. 📝 Basic Usage Examples

Simple Object Serialization

// Header file: Player.h class Player : public CReflectedBase { public: REFL_DECLARE_OBJECT(Player, CReflectedBase); int m_health; float m_speed; std::string m_name; bool m_isAlive; }; // Implementation file: Player.cpp REFL_DEFINE_OBJECT(Player) REFL_DEFINE_INT_MEMBER(Player, m_health), REFL_DEFINE_FLOAT_MEMBER(Player, m_speed), REFL_DEFINE_STRING_MEMBER(Player, m_name), REFL_DEFINE_BOOL_MEMBER(Player, m_isAlive), REFL_DEFINE_END // Usage Player player; player.m_health = 100; player.m_speed = 5.5f; player.m_name = "Hero"; player.m_isAlive = true; // Save to file player.Write("player.json"); // Load from file Player loadedPlayer; loadedPlayer.Read("player.json");

Using Safe Methods with Error Handling

Player player; player.m_name = "Hero"; // Safe write with error handling auto writeResult = player.SafeWrite("player.json"); if (writeResult.IsSuccess()) { std::cout << "Player saved successfully!" << std::endl; } else { const auto& error = writeResult.GetError(); std::cout << "Save failed: " << error.message << std::endl; } // Safe read with error handling Player newPlayer; auto readResult = newPlayer.SafeRead("player.json"); if (readResult.IsSuccess()) { std::cout << "Player loaded: " << newPlayer.m_name << std::endl; } else { const auto& error = readResult.GetError(); std::cout << "Load failed: " << error.message << std::endl; std::cout << "Error category: " << static_cast (error.category) << std::endl; }

6. 🚀 Advanced Usage

Complex Object Hierarchies

// Weapon class class Weapon : public CReflectedBase { public: REFL_DECLARE_OBJECT(Weapon, CReflectedBase); std::string m_name; int m_damage; float m_durability; }; REFL_DEFINE_OBJECT(Weapon) REFL_DEFINE_STRING_MEMBER(Weapon, m_name), REFL_DEFINE_INT_MEMBER(Weapon, m_damage), REFL_DEFINE_FLOAT_MEMBER(Weapon, m_durability), REFL_DEFINE_END // Enhanced Player with complex members class EnhancedPlayer : public CReflectedBase { public: REFL_DECLARE_OBJECT(EnhancedPlayer, CReflectedBase); std::string m_name; Vector4f m_position; Matrix4f m_transform; // Embedded object Weapon m_primaryWeapon; // Smart pointer to object std::unique_ptr m_secondaryWeapon; // Collection of objects std::vector> m_inventory; }; REFL_DEFINE_OBJECT(EnhancedPlayer) REFL_DEFINE_STRING_MEMBER(EnhancedPlayer, m_name), REFL_DEFINE_VECTOR4_MEMBER(EnhancedPlayer, m_position), REFL_DEFINE_MATRIX4_MEMBER(EnhancedPlayer, m_transform), REFL_DEFINE_OBJECT_MEMBER(EnhancedPlayer, m_primaryWeapon), REFL_DEFINE_OBJECT_PTR_MEMBER(EnhancedPlayer, m_secondaryWeapon), REFL_DEFINE_OBJECT_PTR_VECTOR_MEMBER(EnhancedPlayer, m_inventory), REFL_DEFINE_END // Usage EnhancedPlayer player; player.m_name = "Advanced Hero"; player.m_position.set(10.0f, 20.0f, 30.0f, 1.0f); // Initialize embedded weapon player.m_primaryWeapon.m_name = "Sword"; player.m_primaryWeapon.m_damage = 50; // Initialize smart pointer weapon player.m_secondaryWeapon = std::make_unique(); player.m_secondaryWeapon->m_name = "Bow"; player.m_secondaryWeapon->m_damage = 30; // Add items to inventory auto potion = std::make_unique(); potion->m_name = "Health Potion"; player.m_inventory.push_back(std::move(potion)); // Serialize everything player.Write("enhanced_player.json");

Component System Integration

// Component-based entity class GameObject : public CReflectedBase { public: REFL_DECLARE_OBJECT(GameObject, CReflectedBase); std::string m_name; std::vector m_components; }; REFL_DEFINE_OBJECT(GameObject) REFL_DEFINE_STRING_MEMBER(GameObject, m_name), REFL_DEFINE_COMPONENT_PTR_VECTOR_MEMBER(GameObject, m_components), REFL_DEFINE_END // Custom component class HealthComponent : public Component { public: REFL_DECLARE_OBJECT(HealthComponent, Component); int m_maxHealth; int m_currentHealth; }; REFL_DEFINE_OBJECT(HealthComponent) REFL_DEFINE_INT_MEMBER(HealthComponent, m_maxHealth), REFL_DEFINE_INT_MEMBER(HealthComponent, m_currentHealth), REFL_DEFINE_END // Usage with ComponentManager auto* componentManager = Core::CoreSystem::GetComponentManager(); componentManager->RegisterComponentType (HealthComponent::ClassName()); GameObject gameObj; gameObj.m_name = "Player Entity"; // Create and add components auto* healthComp = new HealthComponent(); healthComp->m_maxHealth = 100; healthComp->m_currentHealth = 80; gameObj.m_components.push_back(healthComp); // Serialize with components gameObj.Write("game_object.json");

Validation Data validation errors Invalid reflection map, type mismatch
Memory Memory allocation errors Null pointer, allocation failure

Custom Error Handler

class CustomErrorHandler : public Reflection::IErrorHandler { public: void HandleError(const Reflection::ErrorInfo& error) override { // Log to console std::cout << "[" << error.SeverityToString(error.severity) << "] " << error.CategoryToString(error.category) << ": " << error.message << std::endl; // Log to file if (m_logFile.is_open()) { m_logFile << error.ToString() << std::endl; } // Handle critical errors if (error.severity == Reflection::ErrorSeverity::Critical) { // Perform emergency save or cleanup EmergencyCleanup(); } } bool ShouldThrow(Reflection::ErrorSeverity severity) const override { return severity == Reflection::ErrorSeverity::Critical; } private: std::ofstream m_logFile{"reflection_errors.log"}; void EmergencyCleanup() { // Emergency cleanup code } }; // Set custom error handler Reflection::SetErrorHandler(std::make_unique());

8. 💾 Serialization Formats

Supported File Extensions

Extension Format Parser Characteristics
.json JSON CRFL_JSONParser Human-readable, text-based
.bin, .binary Binary CRFL_BinaryParser Compact, fast, not human-readable
Other/Unknown JSON (default) CRFL_JSONParser Falls back to JSON format
File Extension Detection: The system automatically selects the appropriate parser based on the file extension. Extensions are case-insensitive (.JSON and .json both work).

Format Comparison

// Same object, different formats Player player; player.m_name = "Hero"; player.m_health = 100; // JSON format - human readable player.Write("player.json"); // Result: {"m_name": "Hero", "m_health": 100} // Binary format - compact and fast player.Write("player.bin"); // Result: Binary data (not human readable) // Cross-format compatibility Player jsonPlayer, binPlayer; jsonPlayer.Read("player.json"); // Load from JSON binPlayer.Read("player.bin"); // Load from binary // Both objects will have identical data

9. ✅ Best Practices

🎯 Performance Tips

  • Use Binary format for performance-critical serialization
  • Use JSON format for configuration files and debugging
  • Reserve vector capacity when loading large collections
  • Use SafeRead/SafeWrite for better error handling

⚠️ Common Pitfalls

  • Missing REFL_DEFINE_OBJECT - Objects won't serialize
  • Incorrect member order - Can cause deserialization issues
  • Memory leaks - Use smart pointers for object ownership
  • Ignoring error returns - Always check return values
