GamepadAxis

enum class GamepadAxis : int

Enumeration of all supported gamepad analog axes. Values typically range from -1.0 to 1.0 for sticks, 0.0 to 1.0 for triggers.

LeftX (0): Left stick horizontal axis (-1.0 = left, 1.0 = right)
LeftY (1): Left stick vertical axis (-1.0 = down, 1.0 = up)
RightX (2): Right stick horizontal axis (-1.0 = left, 1.0 = right)
RightY (3): Right stick vertical axis (-1.0 = down, 1.0 = up)
LeftTrigger (4): Left trigger (L2) (0.0 = not pressed, 1.0 = fully pressed)
RightTrigger (5): Right trigger (R2) (0.0 = not pressed, 1.0 = fully pressed)
COUNT (6): Total number of axes
// Example axis usage if (gamepad->GetAxisValue(GamepadAxis::LeftX) > 0.5f) { // Player is pushing left stick to the right player.moveRight(deltaTime); } float triggerValue = gamepad->GetAxisValue(GamepadAxis::RightTrigger); if (triggerValue > 0.1f) { // Player is pressing right trigger weapon.fire(triggerValue); // Intensity based on trigger pressure }

Interfaces

IGamepad Interface

Abstract interface defining the contract for all gamepad implementations. Provides comprehensive access to gamepad state, configuration, and functionality.

Core Functionality

virtual bool Initialize() = 0

Initializes the gamepad. Returns true if successful.

virtual void Update() = 0

Updates the gamepad state. Should be called once per frame.

virtual void Shutdown() = 0

Cleans up gamepad resources and stops input processing.

State Queries

virtual bool IsConnected() const = 0

Returns true if the gamepad is currently connected and functional.

virtual const GamepadState& GetState() const = 0

Returns the current gamepad state.

virtual const GamepadState& GetPreviousState() const = 0

Returns the gamepad state from the previous frame. Useful for detecting state changes.

Gamepad Information

virtual int GetGamepadId() const = 0

Returns the unique identifier for this gamepad.

virtual std::string GetName() const = 0

Returns the human-readable name of the gamepad.

virtual std::string GetGUID() const = 0

Returns the globally unique identifier string for this gamepad type.

Button State Queries

virtual bool IsButtonPressed(GamepadButton button) const = 0

Returns true if the button is currently pressed.

virtual bool IsButtonReleased(GamepadButton button) const = 0

Returns true if the button is currently released (not pressed).

virtual bool WasButtonJustPressed(GamepadButton button) const = 0

Returns true if the button was pressed this frame (transition from released to pressed).

virtual bool WasButtonJustReleased(GamepadButton button) const = 0

Returns true if the button was released this frame (transition from pressed to released).

// Button state examples IGamepad* gamepad = gamepadManager.GetFirstConnectedGamepad(); // Continuous actions (while held) if (gamepad->IsButtonPressed(GamepadButton::A)) { player.jump(); // Keep jumping while A is held } // One-time actions (on press) if (gamepad->WasButtonJustPressed(GamepadButton::Start)) { game.pauseMenu(); // Open pause menu only when Start is first pressed } // Release actions if (gamepad->WasButtonJustReleased(GamepadButton::RightBumper)) { weapon.stopFiring(); // Stop firing when trigger is released }

Axis State Queries

virtual float GetAxisValue(GamepadAxis axis) const = 0

Returns the current value of the specified axis.

virtual float GetAxisDelta(GamepadAxis axis) const = 0

Returns the change in axis value since the previous frame.

// Axis usage examples float leftStickX = gamepad->GetAxisValue(GamepadAxis::LeftX); float rightTrigger = gamepad->GetAxisValue(GamepadAxis::RightTrigger); // Movement with deadzone handling (done automatically by implementation) Vector2 movement(leftStickX, gamepad->GetAxisValue(GamepadAxis::LeftY)); player.move(movement * moveSpeed * deltaTime); // Camera control using axis delta for smooth rotation float cameraDelta = gamepad->GetAxisDelta(GamepadAxis::RightX); camera.yaw(cameraDelta * cameraSpeed * deltaTime);

Callback Types

using GamepadConnectedCallback = std::function<void(int gamepadId)>

Called when a gamepad is connected.

using GamepadDisconnectedCallback = std::function<void(int gamepadId)>

Called when a gamepad is disconnected.

using GamepadButtonCallback = std::function<void(int gamepadId, GamepadButton button, bool pressed)>

Called when a button state changes.

gamepadId: ID of the gamepad that triggered the event
button: Which button changed state
pressed: New button state (true = pressed, false = released)
using GamepadAxisCallback = std::function<void(int gamepadId, GamepadAxis axis, float value)>

Called when an axis value changes significantly.

gamepadId: ID of the gamepad that triggered the event
axis: Which axis changed
value: New axis value
// Callback examples gamepad->SetButtonCallback([](int gamepadId, GamepadButton button, bool pressed) { if (button == GamepadButton::Start && pressed) { std::cout << "Player " << gamepadId << " pressed Start!" << std::endl; game.showPauseMenu(gamepadId); } }); gamepad->SetAxisCallback([](int gamepadId, GamepadAxis axis, float value) { if (axis == GamepadAxis::LeftTrigger && value > 0.5f) { std::cout << "Player " << gamepadId << " pulling left trigger: " << value << std::endl; vehicle.brake(value); } });

Helper Functions

const char* GetButtonName(GamepadButton button)

Returns a human-readable name for the given button.

std::cout << "Pressed: " << GetButtonName(GamepadButton::A) << std::endl; // "Pressed: A"
const char* GetAxisName(GamepadAxis axis)

Returns a human-readable name for the given axis.

std::cout << "Axis: " << GetAxisName(GamepadAxis::LeftX) << std::endl; // "Axis: Left Stick X"
float ApplyDeadzone(float value, float deadzone)

Applies deadzone processing to an axis value. Values within the deadzone are returned as 0.0, while values outside are scaled appropriately.

value: Raw axis value (-1.0 to 1.0)
deadzone: Deadzone threshold (0.0 to 1.0)
Returns: Processed value with deadzone applied
// Manual deadzone application float rawStickValue = 0.08f; float deadzone = 0.1f; float processed = ApplyDeadzone(rawStickValue, deadzone); // Returns 0.0f (within deadzone) rawStickValue = 0.6f; processed = ApplyDeadzone(rawStickValue, deadzone); // Returns ~0.556f (scaled beyond deadzone)
float ApplySensitivity(float value, float sensitivity)

Applies sensitivity scaling to an axis value, clamping the result to valid range.

value: Input axis value
sensitivity: Sensitivity multiplier (typically 0.1 to 3.0)
Returns: Scaled value clamped to [-1.0, 1.0]

Usage Examples

Basic Gamepad Usage

// Simple player movement with gamepad class Player { Vector3 position; float moveSpeed = 5.0f; float jumpHeight = 10.0f; public: void update(IGamepad* gamepad, float deltaTime) { if (!gamepad || !gamepad->IsConnected()) return; // Movement using left stick float moveX = gamepad->GetLeftStickX(); float moveZ = gamepad->GetLeftStickY(); Vector3 movement(moveX, 0, moveZ); if (movement.magnitude() > 0.1f) { // Small deadzone check position += movement * moveSpeed * deltaTime; } // Jump with A button (one-time action) if (gamepad->WasButtonJustPressed(GamepadButton::A)) { jump(jumpHeight); } // Run while holding right bumper float currentSpeed = moveSpeed; if (gamepad->IsButtonPressed(GamepadButton::RightBumper)) { currentSpeed *= 2.0f; // Double speed while running } } };

Camera Control

// FPS camera control with gamepad class FPSCamera { Vector3 position; float yaw = 0.0f; float pitch = 0.0f; float mouseSensitivity = 2.0f; public: void update(IGamepad* gamepad, float deltaTime) { if (!gamepad || !gamepad->IsConnected()) return; // Camera rotation using right stick float rightStickX = gamepad->GetRightStickX(); float rightStickY = gamepad->GetRightStickY(); // Apply sensitivity and delta time yaw += rightStickX * mouseSensitivity * deltaTime; pitch -= rightStickY * mouseSensitivity * deltaTime; // Invert Y-axis // Clamp pitch to prevent over-rotation pitch = std::clamp(pitch, -89.0f, 89.0f); // Optional: Variable sensitivity based on trigger pressure float lookSpeed = mouseSensitivity; if (gamepad->GetRightTrigger() > 0.1f) { // Precision aiming when right trigger is held lookSpeed *= 0.3f; } } };

Menu Navigation

// Menu system with gamepad navigation class GameMenu { int selectedIndex = 0; std::vector menuItems; float navigationCooldown = 0.0f; public: void update(IGamepad* gamepad, float deltaTime) { if (!gamepad || !gamepad->IsConnected()) return; navigationCooldown -= deltaTime; // Navigate with D-Pad or left stick bool moveUp = gamepad->WasButtonJustPressed(GamepadButton::DPadUp) || (gamepad->GetLeftStickY() > 0.7f && navigationCooldown <= 0.0f); bool moveDown = gamepad->WasButtonJustPressed(GamepadButton::DPadDown) || (gamepad->GetLeftStickY() < -0.7f && navigationCooldown <= 0.0f); if (moveUp) { selectedIndex = (selectedIndex - 1 + menuItems.size()) % menuItems.size(); navigationCooldown = 0.2f; // Prevent rapid navigation playSound("menu_move"); } else if (moveDown) { selectedIndex = (selectedIndex + 1) % menuItems.size(); navigationCooldown = 0.2f; playSound("menu_move"); } // Select with A button if (gamepad->WasButtonJustPressed(GamepadButton::A)) { menuItems[selectedIndex].activate(); playSound("menu_select"); } // Back with B button if (gamepad->WasButtonJustPressed(GamepadButton::B)) { goBack(); playSound("menu_back"); } } };

Driving Game

// Racing game vehicle control class Vehicle { float speed = 0.0f; float steering = 0.0f; float maxSpeed = 100.0f; bool isBraking = false; public: void update(IGamepad* gamepad, float deltaTime) { if (!gamepad || !gamepad->IsConnected()) return; // Steering with left stick float steerInput = gamepad->GetLeftStickX(); steering = steerInput; // -1.0 = full left, 1.0 = full right // Acceleration and braking with triggers float accelerate = gamepad->GetRightTrigger(); // 0.0 to 1.0 float brake = gamepad->GetLeftTrigger(); // 0.0 to 1.0 // Apply acceleration if (accelerate > 0.1f) { speed += accelerate * 50.0f * deltaTime; // Acceleration rate speed = std::min(speed, maxSpeed); } // Apply braking if (brake > 0.1f) { isBraking = true; speed -= brake * 80.0f * deltaTime; // Braking is stronger than acceleration speed = std::max(speed, 0.0f); // Vibration feedback for braking if (gamepad->SupportsVibration()) { gamepad->SetVibration(brake * 0.3f, brake * 0.5f); } } else { isBraking = false; // Natural deceleration speed -= 20.0f * deltaTime; speed = std::max(speed, 0.0f); gamepad->StopVibration(); } // Handbrake with A button if (gamepad->IsButtonPressed(GamepadButton::A)) { speed *= 0.95f; // Rapid deceleration // Stronger vibration for handbrake if (gamepad->SupportsVibration()) { gamepad->SetVibration(0.6f, 0.8f); } } } };

Event-Driven Input Handling

// Event-driven input system class InputEventHandler { public: void setupGamepadEvents(IGamepad* gamepad) { // Button event handling gamepad->SetButtonCallback([this](int gamepadId, GamepadButton button, bool pressed) { handleButtonEvent(gamepadId, button, pressed); }); // Axis event handling gamepad->SetAxisCallback([this](int gamepadId, GamepadAxis axis, float value) { handleAxisEvent(gamepadId, axis, value); }); } private: void handleButtonEvent(int gamepadId, GamepadButton button, bool pressed) { if (!pressed) return; // Only handle button presses switch (button) { case GamepadButton::Start: game->togglePause(); break; case GamepadButton::Back: game->showInventory(); break; case GamepadButton::Y: player->useItem(); break; case GamepadButton::LeftBumper: player->previousWeapon(); break; case GamepadButton::RightBumper: player->nextWeapon(); break; } } void handleAxisEvent(int gamepadId, GamepadAxis axis, float value) { switch (axis) { case GamepadAxis::LeftTrigger: if (value > 0.8f) { weapon->chargeAttack(value); } break; case GamepadAxis::RightTrigger: if (value > 0.1f) { weapon->fire(value); } else { weapon->stopFiring(); } break; } } };

Concrete Implementations

GLFWGamepad

GLFWGamepad - Implementation using GLFW for cross-platform gamepad support. Supports XInput controllers, PlayStation controllers, and other common gamepads through GLFW's gamepad API.

  • Platform Support: Windows, macOS, Linux
  • Controller Support: Xbox 360/One/Series, PlayStation 3/4/5, Nintendo Switch Pro Controller
  • Features: Automatic button mapping, hotplug support, vibration simulation

MKBGamepad

MKBGamepad - Mouse and Keyboard implementation that emulates gamepad behavior. Useful for development and providing gamepad-like interface for keyboard players.

  • Features: Customizable key mapping, mouse-as-analog-stick, configurable sensitivity
  • Use Cases: Development testing, accessibility, mixed input scenarios
  • Key Mappings: WASD for left stick, mouse for right stick, Space for A button, etc.

GamepadManager

GamepadManager - High-level manager that handles multiple gamepads, automatic detection, and input routing to handlers.

  • Features: Multi-gamepad support, hotplug detection, input handler stack, global events
  • Input Handlers: Priority-based input processing with IGamepadInputHandler
  • Management: Automatic cleanup, connection monitoring, configuration management

Best Practices

Input Handling

  • Frame-based updates: Always call Update() once per frame before querying state
  • Connection checking: Always check IsConnected() before using gamepad input
  • Just-pressed vs pressed: Use WasButtonJustPressed() for one-time actions, IsButtonPressed() for continuous actions
  • Null checking: Always verify gamepad pointers are not null before use

Configuration and Accessibility

  • Configurable deadzones: Allow users to adjust deadzones for accessibility and worn controllers
  • Sensitivity options: Provide sensitivity settings for different player preferences
  • Button remapping: Consider allowing users to remap buttons for accessibility
  • Multiple input methods: Support both gamepad and keyboard/mouse simultaneously

Performance Considerations

  • Event callbacks: Keep callback functions lightweight to avoid frame rate issues
  • State caching: Cache frequently accessed values rather than calling methods repeatedly
  • Vibration cleanup: Always stop vibration when appropriate to avoid draining battery
  • Connection polling: Use GamepadManager for efficient connection monitoring rather than polling individual gamepads

Cross-Platform Compatibility

  • Button conventions: Be aware of different button naming conventions (Xbox vs PlayStation)
  • Axis directions: Some platforms may have different axis orientations
  • Vibration support: Not all platforms/controllers support vibration
  • Device detection: Controller hotplug behavior varies between platforms

User Experience

  • Visual feedback: Show on-screen prompts that match the connected controller type
  • Graceful degradation: Handle controller disconnection gracefully
  • Input buffering: Consider buffering critical inputs to handle brief disconnections
  • Mixed input: Allow seamless switching between gamepad and keyboard/mouse
