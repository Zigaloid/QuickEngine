1. 🌟 Overview

The CoreSystem is the central orchestrator for all engine systems in the C++20 game engine. It implements a singleton pattern that manages the initialization, coordination, and shutdown of core subsystems including file management, resource loading, component management, job scheduling, and system coordination.

Key Features:
  • ✅ Centralized system management and initialization
  • ✅ Singleton pattern for global access
  • ✅ Dependency management between systems
  • ✅ Exception-safe initialization and cleanup
  • ✅ Comprehensive system status monitoring
  • ✅ Integrated debug logging with CoreDebug channel
  • ✅ Thread-safe system access
  • ✅ Graceful error handling and recovery

5. 🔄 System Lifecycle

Initialization Sequence

// Initialization order (dependencies matter!) 1. FileSystemManager → Base file operations 2. ResourceManager → Asset loading (needs FileSystem) 3. JobSystem → Task scheduling 4. ComponentManager → Component pools and lifecycle 5. ComponentSystemScheduler → Async updates (needs Components + Jobs)

Shutdown Sequence

// Shutdown order (reverse of initialization) 1. ComponentSystemScheduler → Stop async updates first 2. ComponentManager → Clean up components 3. JobSystem → Finish running tasks 4. ResourceManager → Unload resources 5. FileSystemManager → Close file handles

11. ✅ Best Practices

🎯 Initialization Guidelines

  • Initialize early - Call CoreSystem::Initialize() before using any subsystems
  • Check return values - Always verify initialization success
  • Use RAII patterns - Automatic cleanup prevents resource leaks
  • Handle exceptions - Wrap initialization in try-catch blocks
  • Verify systems - Check system availability before use

✅ Good Practices

// ✅ Good: Safe initialization with verification bool InitializeEngine() { if (!Core::CoreSystem::Initialize()) { return false; } // Verify critical systems if (!Core::CoreSystem::GetFileSystemManager() || !Core::CoreSystem::GetJobSystem()) { Core::CoreSystem::Shutdown(); return false; } return true; } // ✅ Good: Safe system access void SafeFileOperation() { auto* fs = Core::CoreSystem::GetFileSystemManager(); if (!fs) { CoreDebug.warning("FileSystem not available\n"); return; } auto result = fs->ReadAllText("config.json"); if (result.IsSuccess()) { ProcessConfig(result.GetValue()); } } // ✅ Good: Proper shutdown sequence void CleanShutdown() { // Wait for pending work auto* jobSystem = Core::CoreSystem::GetJobSystem(); if (jobSystem) { jobSystem->waitForAll(); } // Print final status Core::CoreSystem::PrintSystemStatus(); // Shutdown Core::CoreSystem::Shutdown(); } // ✅ Good: RAII management class EngineManager { bool m_initialized = false; public: EngineManager() { m_initialized = Core::CoreSystem::Initialize(); if (!m_initialized) { throw std::runtime_error("Failed to initialize"); } } ~EngineManager() { if (m_initialized) { Core::CoreSystem::Shutdown(); } } };

⚠️ Common Pitfalls

// ❌ Poor: Not checking initialization Core::CoreSystem::Initialize(); // What if this fails? auto* fs = Core::CoreSystem::GetFileSystemManager(); // Could be null! // ❌ Poor: Using systems before initialization auto* jobSystem = Core::CoreSystem::GetJobSystem(); // Null pointer! jobSystem->submitJob([](){}); // Crash! // ❌ Poor: Forgetting to shutdown int main() { Core::CoreSystem::Initialize(); RunApplication(); // Missing Core::CoreSystem::Shutdown() - resource leak! return 0; } // ❌ Poor: Not handling exceptions Core::CoreSystem::Initialize(); // Could throw // No exception handling // ❌ Poor: Ignoring system status void Update() { auto* scheduler = Core::CoreSystem::GetJobSystemScheduler(); scheduler->UpdateAllAsync(0.016); // What if scheduler is null? }

12. 🔧 Troubleshooting

Common Issues and Solutions

Issue Symptoms Solution
Initialization fails Initialize() returns false Check CoreDebug output for specific failure reasons
Null pointer access Crash when calling Get*() methods Verify CoreSystem::IsInitialized() before system access
System not available Get*() methods return nullptr Ensure initialization completed successfully
Memory leaks Increasing memory usage Call Shutdown() before application exit
Shutdown hangs Application doesn't exit Wait for pending jobs before shutdown
