Quaternion Class Documentation

Class Overview

class Quaternion

Purpose: A mathematical representation of rotations in 3D space using quaternions. Provides gimbal-lock-free rotation operations, smooth interpolation, and efficient composition of rotations.

Header: Core/Quaternion.h

Dependencies: Vector3f, Matrix4f, cmath

Mathematical Foundation

A quaternion is represented as q = w + xi + yj + zk where:

w
(scalar)
x
(i)
y
(j)
z
(k)

w = scalar part, (x, y, z) = vector part

Constructors

Quaternion()

Default constructor. Creates an identity quaternion representing no rotation.

Quaternion q; // Creates (1, 0, 0, 0) - identity quaternion
Quaternion(float w, float x, float y, float z)

Constructs a quaternion with specified components.

w: Scalar component
x: i component of vector part
y: j component of vector part
z: k component of vector part
Quaternion q(0.707f, 0.707f, 0.0f, 0.0f); // 90-degree rotation around X-axis
Quaternion(const Vector3f& axis, float angleRad)

Constructs a quaternion from axis-angle representation.

axis: Rotation axis (will be normalized)
angleRad: Rotation angle in radians
Vector3f axis(0, 1, 0); // Y-axis float angle = M_PI / 4; // 45 degrees Quaternion q(axis, angle); // 45-degree rotation around Y-axis
Quaternion(float pitch, float yaw, float roll)

Constructs a quaternion from Euler angles (in radians).

pitch: Rotation around X-axis (radians)
yaw: Rotation around Y-axis (radians)
roll: Rotation around Z-axis (radians)
float pitch = M_PI / 6; // 30 degrees float yaw = M_PI / 4; // 45 degrees float roll = 0.0f; // 0 degrees Quaternion q(pitch, yaw, roll);

Arithmetic Operations

Quaternion operator+(const Quaternion& other) const
Quaternion& operator+=(const Quaternion& other)

Quaternion addition (component-wise). Note: The sum of two unit quaternions is not necessarily a unit quaternion.

Quaternion q1(1, 0, 0, 0); Quaternion q2(0, 1, 0, 0); Quaternion sum = q1 + q2; // (1, 1, 0, 0)
Quaternion operator*(const Quaternion& other) const
Quaternion& operator*=(const Quaternion& other)

Quaternion multiplication (Hamilton product). This composes rotations: q1 * q2 applies q2 first, then q1.

Note: Quaternion multiplication is not commutative! q1 * q2 ≠ q2 * q1
Quaternion rotX = Quaternion::rotationX(M_PI / 2); // 90° around X Quaternion rotY = Quaternion::rotationY(M_PI / 2); // 90° around Y Quaternion combined = rotX * rotY; // Apply Y rotation, then X rotation
Quaternion operator*(float scalar) const
Quaternion& operator*=(float scalar)

Scalar multiplication. Multiplies all components by the scalar.

Quaternion q(1, 0, 0, 0); Quaternion scaled = q * 2.0f; // (2, 0, 0, 0)

Quaternion Operations

float dot(const Quaternion& other) const

Calculates the dot product of two quaternions. Useful for measuring similarity between rotations.

Quaternion q1(1, 0, 0, 0); Quaternion q2(0.707f, 0.707f, 0, 0); float similarity = q1.dot(q2); // Returns ~0.707
float magnitude() const
float magnitudeSquared() const

Returns the magnitude (norm) of the quaternion. For unit quaternions, this should be 1.0.

Quaternion q(0.5f, 0.5f, 0.5f, 0.5f); float mag = q.magnitude(); // Returns 1.0f float magSq = q.magnitudeSquared(); // Returns 1.0f (more efficient)
Quaternion& normalize()
Quaternion normalized() const

Normalizes the quaternion to unit length. Essential for representing valid rotations.

Warning: These methods throw std::runtime_error if the quaternion is zero.
Quaternion q(2, 2, 2, 2); Quaternion unit = q.normalized(); // Returns (0.5, 0.5, 0.5, 0.5) q.normalize(); // q is now (0.5, 0.5, 0.5, 0.5)
Quaternion conjugate() const

Returns the quaternion conjugate (w, -x, -y, -z). For unit quaternions, this represents the inverse rotation.

Quaternion q(0.707f, 0.707f, 0, 0); // 90° around X Quaternion conj = q.conjugate(); // Represents -90° around X
Quaternion inverse() const

Returns the quaternion inverse. For unit quaternions, this is equivalent to the conjugate.

Warning: Throws std::runtime_error if the quaternion is zero.
Quaternion q(0.707f, 0.707f, 0, 0); Quaternion inv = q.inverse(); Quaternion identity = q * inv; // Should be approximately (1, 0, 0, 0)

Conversion Methods

void fromAxisAngle(const Vector3f& axis, float angleRad)
void toAxisAngle(Vector3f& axis, float& angleRad) const

Convert between quaternion and axis-angle representation.

// From axis-angle to quaternion Vector3f axis(0, 1, 0); float angle = M_PI / 3; // 60 degrees Quaternion q; q.fromAxisAngle(axis, angle); // From quaternion to axis-angle Vector3f extractedAxis; float extractedAngle; q.toAxisAngle(extractedAxis, extractedAngle);
void fromEulerAngles(float pitch, float yaw, float roll)
Vector3f toEulerAngles() const

Convert between quaternion and Euler angles (in radians).

Note: Euler angle conversion can suffer from gimbal lock. Use quaternions for intermediate calculations when possible.
// From Euler angles to quaternion Quaternion q; q.fromEulerAngles(M_PI/6, M_PI/4, M_PI/3); // 30°, 45°, 60° // From quaternion to Euler angles Vector3f eulerAngles = q.toEulerAngles(); float pitch = eulerAngles.getX(); float yaw = eulerAngles.getY(); float roll = eulerAngles.getZ();
Matrix4f toMatrix() const
static Quaternion fromMatrix(const Matrix4f& matrix)

Convert between quaternion and 4x4 rotation matrix.

// Convert quaternion to matrix Quaternion q = Quaternion::rotationY(M_PI / 4); Matrix4f rotMatrix = q.toMatrix(); // Convert matrix to quaternion Quaternion q2 = Quaternion::fromMatrix(rotMatrix);

Vector Operations

Vector3f rotateVector(const Vector3f& vector) const
Vector3f operator*(const Vector3f& vector) const

Rotates a 3D vector by this quaternion. The operator* provides convenient syntax.

Vector3f forward(0, 0, 1); Quaternion q = Quaternion::rotationY(M_PI / 2); // 90° around Y-axis Vector3f rotated1 = q.rotateVector(forward); // Method call Vector3f rotated2 = q * forward; // Operator syntax // Both return (-1, 0, 0) - forward vector rotated to point left
Vector3f getForward() const
Vector3f getRight() const
Vector3f getUp() const

Gets the local coordinate axes after applying this rotation.

Quaternion q = Quaternion::rotationY(M_PI / 2); // 90° around Y Vector3f forward = q.getForward(); // (-1, 0, 0) Vector3f right = q.getRight(); // (0, 0, 1) Vector3f up = q.getUp(); // (0, 1, 0)

Interpolation Methods

static Quaternion slerp(const Quaternion& a, const Quaternion& b, float t)

Spherical Linear Interpolation. Provides smooth, constant-speed interpolation between two rotations.

a: Start quaternion
b: End quaternion
t: Interpolation parameter (0.0 to 1.0)
Quaternion start = Quaternion::identity(); Quaternion end = Quaternion::rotationY(M_PI); // 180° rotation // Smooth rotation animation for (float t = 0.0f; t <= 1.0f; t += 0.1f) { Quaternion current = Quaternion::slerp(start, end, t); // Use 'current' for animation frame }
static Quaternion nlerp(const Quaternion& a, const Quaternion& b, float t)

Normalized Linear Interpolation. Faster than SLERP but with non-constant angular velocity.

Performance: NLERP is faster than SLERP and often visually acceptable for short interpolations.
Quaternion start = Quaternion::identity(); Quaternion end = Quaternion::rotationX(M_PI / 4); // Fast interpolation (good for small angle differences) Quaternion interpolated = Quaternion::nlerp(start, end, 0.5f);

Creates quaternions for rotation around principal axes.

Quaternion pitchDown = Quaternion::rotationX(-M_PI / 6); // -30° pitch Quaternion turnRight = Quaternion::rotationY(M_PI / 4); // 45° yaw Quaternion rollLeft = Quaternion::rotationZ(M_PI / 8); // 22.5° roll
static Quaternion fromToRotation(const Vector3f& from, const Vector3f& to)

Creates a quaternion that rotates vector 'from' to align with vector 'to'.

Vector3f currentDir(1, 0, 0); // Pointing right Vector3f targetDir(0, 0, 1); // Want to point forward Quaternion rotation = Quaternion::fromToRotation(currentDir, targetDir); Vector3f result = rotation * currentDir; // Now points forward
static Quaternion lookRotation(const Vector3f& forward, const Vector3f& up = Vector3f(0, 1, 0))

Creates a rotation that looks in the specified direction. Useful for cameras and object orientation.

Vector3f lookDirection = (target - position).normalized(); Quaternion cameraOrientation = Quaternion::lookRotation(lookDirection); // Apply to camera camera.setOrientation(cameraOrientation);

Utility Functions

bool isUnit() const

Checks if the quaternion is normalized (unit length). Essential for valid rotations.

Quaternion q(0.707f, 0.707f, 0, 0); if (q.isUnit()) { std::cout << "Valid rotation quaternion" << std::endl; }
bool isZero() const

Checks if the quaternion is approximately zero.

Quaternion q(0.0000001f, 0, 0, 0); if (q.isZero()) { std::cout << "Quaternion is effectively zero" << std::endl; }

Usage Examples

Basic Rotation Operations

// Create rotations Quaternion rotX = Quaternion::rotationX(M_PI / 4); // 45° around X-axis Quaternion rotY = Quaternion::rotationY(M_PI / 3); // 60° around Y-axis // Combine rotations (order matters!) Quaternion combined = rotY * rotX; // Apply X rotation first, then Y // Rotate a vector Vector3f point(1, 0, 0); Vector3f rotatedPoint = combined * point; std::cout << "Original: " << point << std::endl; std::cout << "Rotated: " << rotatedPoint << std::endl;

Camera System Integration

class FPSCamera { Vector3f position; Quaternion orientation; public: void rotate(float deltaYaw, float deltaPitch) { // Yaw around world Y-axis Quaternion yawRotation = Quaternion::rotationY(deltaYaw); orientation = yawRotation * orientation; // Pitch around local X-axis Quaternion pitchRotation = Quaternion::rotationX(deltaPitch); orientation = orientation * pitchRotation; orientation.normalize(); } void moveForward(float distance) { Vector3f forward = orientation.getForward(); position += forward * distance; } Matrix4f getViewMatrix() const { Matrix4f rotation = orientation.conjugate().toMatrix(); Matrix4f translation = Matrix4f::translation(-position.getX(), -position.getY(), -position.getZ()); return rotation * translation; } };

Animation and Interpolation

// Smooth rotation animation between two orientations class RotationAnimator { Quaternion startRotation; Quaternion endRotation; float duration; float currentTime; public: RotationAnimator(const Quaternion& start, const Quaternion& end, float dur) : startRotation(start), endRotation(end), duration(dur), currentTime(0) {} Quaternion update(float deltaTime) { currentTime += deltaTime; float t = std::min(currentTime / duration, 1.0f); // Use SLERP for smooth interpolation return Quaternion::slerp(startRotation, endRotation, t); } bool isComplete() const { return currentTime >= duration; } }; // Usage in game loop RotationAnimator animator(Quaternion::identity(), Quaternion::rotationY(M_PI), 2.0f); // 2-second rotation while (!animator.isComplete()) { Quaternion currentRotation = animator.update(deltaTime); object.setRotation(currentRotation); render(); }

Physics Integration

// Rigid body rotation using quaternions class RigidBody { Vector3f position; Vector3f velocity; Quaternion orientation; Vector3f angularVelocity; // Axis of rotation * angular speed public: void update(float deltaTime) { // Update position position += velocity * deltaTime; // Update rotation using angular velocity if (!angularVelocity.isZero()) { float angularSpeed = angularVelocity.magnitude(); Vector3f axis = angularVelocity.normalized(); Quaternion deltaRotation(axis, angularSpeed * deltaTime); orientation = deltaRotation * orientation; orientation.normalize(); } } void applyTorque(const Vector3f& torque, float deltaTime) { // Simple integration of angular velocity angularVelocity += torque * deltaTime; } Matrix4f getTransformMatrix() const { Matrix4f rotation = orientation.toMatrix(); Matrix4f translation = Matrix4f::translation(position.getX(), position.getY(), position.getZ()); return translation * rotation; } };

Converting Between Representations

// Working with different rotation representations void demonstrateConversions() { // Start with Euler angles (degrees converted to radians) float pitchDeg = 30.0f, yawDeg = 45.0f, rollDeg = 60.0f; float pitch = pitchDeg * M_PI / 180.0f; float yaw = yawDeg * M_PI / 180.0f; float roll = rollDeg * M_PI / 180.0f; // Convert to quaternion Quaternion q; q.fromEulerAngles(pitch, yaw, roll); // Convert to axis-angle Vector3f axis; float angle; q.toAxisAngle(axis, angle); std::cout << "Axis: " << axis << ", Angle: " << (angle * 180.0f / M_PI) << "°" << std::endl; // Convert to rotation matrix Matrix4f rotMatrix = q.toMatrix(); // Convert back to quaternion from matrix Quaternion q2 = Quaternion::fromMatrix(rotMatrix); // Verify they're the same (accounting for potential sign flip) float dot = q.dot(q2); bool equivalent = (std::abs(dot) > 0.999f); std::cout << "Conversion successful: " << equivalent << std::endl; }

Object Alignment

// Align one object to look at another class GameObject { Vector3f position; Quaternion orientation; public: void lookAt(const Vector3f& target, const Vector3f& worldUp = Vector3f(0, 1, 0)) { Vector3f direction = (target - position).normalized(); // Create rotation to look in that direction orientation = Quaternion::lookRotation(direction, worldUp); } void smoothLookAt(const Vector3f& target, float turnSpeed, float deltaTime) { Vector3f direction = (target - position).normalized(); Quaternion targetRotation = Quaternion::lookRotation(direction); // Smoothly interpolate to target rotation float t = std::min(turnSpeed * deltaTime, 1.0f); orientation = Quaternion::slerp(orientation, targetRotation, t); } // Get the forward direction vector Vector3f getForwardDirection() const { return orientation.getForward(); } };

Best Practices

Normalization

  • Always normalize: Keep quaternions normalized when representing rotations
  • Accumulation errors: Renormalize periodically during long sequences of operations
  • Check validity: Use isUnit() to verify quaternion validity

Performance Optimization

  • Use conjugate for unit quaternions: For unit quaternions, conjugate() is faster than inverse()
  • NLERP vs SLERP: Use NLERP for small angle differences, SLERP for accurate interpolation
  • Avoid Euler conversions: Minimize conversions to/from Euler angles in inner loops
  • Cache results: Store converted matrices/vectors when used repeatedly

Common Pitfalls

  • Order of operations: Remember that quaternion multiplication is not commutative
  • Double cover: q and -q represent the same rotation; handle this in interpolation
  • Gimbal lock: Euler angles can still cause issues even when using quaternions for storage
  • Precision loss: Repeated operations can accumulate floating-point errors

Mathematical Properties

  • Unit quaternions: For rotations, |q| = 1
  • Inverse: For unit quaternions, q⁻¹ = q* (conjugate)
  • Composition: q₂ * q₁ applies rotation q₁ first, then q₂
  • Identity: (1, 0, 0, 0) represents no rotation
  • Double cover: q and -q represent the same rotation

Integration with Graphics APIs

  • OpenGL: Use toMatrix() to convert to 4x4 matrices for shaders
  • Camera systems: Use conjugate for view matrix conversion
  • Animation systems: SLERP provides smooth, professional-quality rotation animation
  • Physics engines: Most physics engines accept quaternions directly for orientation
