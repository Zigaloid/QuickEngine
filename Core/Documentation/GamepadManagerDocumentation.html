GamepadManager Class Documentation

Class Overview

class GamepadManager

Purpose: A comprehensive manager for handling multiple gamepads with automatic detection, input routing through priority-based handlers, and centralized configuration. Provides hotplug support, event callbacks, and utility functions for multi-gamepad applications.

Namespace: Input

Header: Core/GamepadManager.h

Implementation: Core/GamepadManager.cpp

Architecture Overview

The GamepadManager implements a layered input processing system with priority-based handlers:

Input Handler Processing Flow

UI Handler
(Priority: 100)
Menu Handler
(Priority: 50)
Game Handler
(Priority: 10)

↓ Input flows through handlers in priority order ↓

Higher priority handlers can consume input to prevent lower priority handlers from processing it.

Usage Examples

Basic Setup and Usage

// Initialize gamepad system GamepadConfig config; config.deadzone = 0.15f; config.maxControllers = 4; GamepadManager manager(config); if (!manager.Initialize()) { std::cerr << "Failed to initialize gamepad manager" << std::endl; return -1; } // Main game loop while (gameRunning) { double deltaTime = timer.getDeltaTime(); manager.Update(deltaTime); // Simple single-player input IGamepad* controller = manager.GetFirstConnectedGamepad(); if (controller) { if (controller->WasButtonJustPressed(GamepadButton::Start)) { game.togglePause(); } Vector2 movement(controller->GetLeftStickX(), controller->GetLeftStickY()); player.move(movement * deltaTime); } // ... rest of game logic } manager.Shutdown();

Multi-Player Game with Input Handlers

// Multi-player game with layered input handling class MultiPlayerGame { GamepadManager gamepadManager_; std::shared_ptr menuHandler_; std::shared_ptr gameHandler_; public: void initialize() { // Setup gamepad manager GamepadConfig config; config.maxControllers = 4; gamepadManager_ = GamepadManager(config); gamepadManager_.Initialize(); // Create input handlers menuHandler_ = std::make_shared ("MenuSystem", 100); gameHandler_ = std::make_shared ("GamePlay", 10); // Setup connection callbacks gamepadManager_.SetConnectedCallback([this](int gamepadId) { onPlayerJoined(gamepadId); }); gamepadManager_.SetDisconnectedCallback([this](int gamepadId) { onPlayerLeft(gamepadId); }); // Start with menu input only gamepadManager_.AddInputHandler(menuHandler_); } void startGame() { // Add game input handler when starting gameplay gamepadManager_.AddInputHandler(gameHandler_); menuHandler_->SetActive(false); // Disable menu input } void showMenu() { // Re-enable menu input, disable game input menuHandler_->SetActive(true); gameHandler_->SetActive(false); } void update(double deltaTime) { gamepadManager_.Update(deltaTime); // Check if any player wants to pause if (gamepadManager_.IsAnyButtonJustPressed(GamepadButton::Start)) { showMenu(); } } private: void onPlayerJoined(int gamepadId) { std::cout << "Player " << gamepadId << " joined!" << std::endl; ui_.showPlayerJoinedNotification(gamepadId); // Configure controller for this player IGamepad* gamepad = gamepadManager_.GetGamepad(gamepadId); if (gamepad && gamepad->SupportsVibration()) { gamepad->SetVibration(0.5f, 0.5f); // Welcome vibration // Stop vibration after 200ms scheduler_.scheduleAfter(200ms, [gamepad]() { gamepad->StopVibration(); }); } } void onPlayerLeft(int gamepadId) { std::cout << "Player " << gamepadId << " disconnected!" << std::endl; game_.pauseForReconnection(gamepadId); } };

Custom Input Handler Implementation

// Example custom input handler for a racing game class RacingInputHandler : public GamepadInputHandler { RacingGame* game_; public: RacingInputHandler(RacingGame* game, int priority = 10) : GamepadInputHandler("RacingInput", priority), game_(game) {} bool HandleButtonInput(int gamepadId, GamepadButton button, bool pressed, bool justPressed, bool justReleased) override { if (!IsActive()) return false; Player* player = game_->getPlayer(gamepadId); if (!player) return false; switch (button) { case GamepadButton::A: if (justPressed) { player->startHandbrake(); return true; // Consume input } else if (justReleased) { player->stopHandbrake(); return true; } break; case GamepadButton::Start: if (justPressed) { game_->showPauseMenu(gamepadId); return true; // Prevent other handlers from processing } break; case GamepadButton::Back: if (justPressed) { player->lookBehind(!pressed); return true; } break; } return false; // Let other handlers process unhandled input } bool HandleAxisInput(int gamepadId, GamepadAxis axis, float value, float delta) override { if (!IsActive()) return false; Player* player = game_->getPlayer(gamepadId); if (!player) return false; switch (axis) { case GamepadAxis::LeftX: player->setSteering(value); return true; case GamepadAxis::RightTrigger: player->setAcceleration(value); return true; case GamepadAxis::LeftTrigger: player->setBraking(value); return true; case GamepadAxis::RightY: // Camera pitch for looking around if (std::abs(value) > 0.1f) { player->adjustCameraPitch(value * 2.0f); return true; } break; } return false; } void OnGamepadConnected(int gamepadId) override { std::cout << "Racing controller " << gamepadId << " connected" << std::endl; game_->addPlayer(gamepadId); } void OnGamepadDisconnected(int gamepadId) override { std::cout << "Racing controller " << gamepadId << " disconnected" << std::endl; game_->removePlayer(gamepadId); } };

Best Practices

Initialization and Lifecycle

  • Initialize early: Call Initialize() during application startup, before the main loop
  • Update regularly: Call Update() once per frame in your main loop
  • Handle shutdown: Explicit Shutdown() call is optional but good practice for clarity
  • Check initialization: Always check the return value of Initialize()

Input Handler Design

  • Priority hierarchy: Use clear priority levels (e.g., UI=100, Menu=50, Game=10)
  • Consume input appropriately: Return true from handlers only when input should be consumed
  • State management: Use IsActive() to enable/disable handlers instead of removing them
  • Handler lifetime: Store handlers as shared_ptr to manage lifetime properly

Multi-Player Considerations

  • Hotplug handling: Always implement connection/disconnection callbacks for robust multi-player
  • Player assignment: Maintain mapping between gamepad IDs and player slots
  • Graceful degradation: Handle mid-game disconnections with pause/reconnect logic
  • Configuration per player: Allow per-gamepad configuration for accessibility

Performance and Architecture

  • Handler efficiency: Keep input handlers lightweight to avoid frame rate impact
  • Auto-detection overhead: Disable auto-detection if managing gamepads manually
  • Memory management: The manager uses smart pointers for automatic resource cleanup
  • Thread safety: The manager is not thread-safe; use from main thread only

Integration Patterns

  • Singleton usage: Use GlobalGamepadManager for simple applications, instance-based for complex ones
  • State machines: Integrate handler activation with game state transitions
  • Event systems: Use callbacks to integrate with broader event/messaging systems
  • Configuration systems: Store and load gamepad configurations with player profiles
