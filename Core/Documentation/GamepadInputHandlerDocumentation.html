GamepadInputHandler Documentation

Module Overview

Purpose: The GamepadInputHandler module provides a priority-based input processing system for gamepad events. It enables layered input handling where different game systems (UI, menus, gameplay) can process input in a controlled order with the ability to consume input to prevent lower-priority handlers from processing it.

Namespace: Input

Header: Core/GamepadInputHandler.h

Input Processing Architecture

The system uses a chain-of-responsibility pattern where input events flow through handlers in priority order:

Input Event Flow

(Button/Axis Events)

↑ Handlers can return true to consume input and stop the chain ↑

IGamepadInputHandler Interface

class IGamepadInputHandler

Purpose: Abstract interface defining the contract for gamepad input processing handlers. Handlers are organized in a priority-based stack where higher priority handlers process input first and can consume it to prevent lower priority handlers from receiving it.

Pure Virtual Methods (Must Override)

virtual bool HandleButtonInput(int gamepadId, GamepadButton button, bool pressed, bool justPressed, bool justReleased) = 0

Processes button input events from gamepads. This is the core method for handling button-based interactions.

gamepadId: Unique identifier of the gamepad that generated the event
button: Which button triggered the event (see GamepadButton enum)
pressed: Current state of the button (true = pressed, false = released)
justPressed: True if the button was just pressed this frame (transition from released to pressed)
justReleased: True if the button was just released this frame (transition from pressed to released)
Returns: true if the input was handled and should not be passed to lower priority handlers, false to continue processing
// Example button handling for a menu system bool MenuInputHandler::HandleButtonInput(int gamepadId, GamepadButton button, bool pressed, bool justPressed, bool justReleased) { if (!IsActive()) return false; // Only process button presses for menu navigation if (justPressed) { switch (button) { case GamepadButton::A: menu->selectCurrentItem(); return true; // Consume input - don't let game handler process it case GamepadButton::B: menu->goBack(); return true; // Consume input case GamepadButton::DPadUp: menu->navigateUp(); return true; // Consume input case GamepadButton::DPadDown: menu->navigateDown(); return true; // Consume input case GamepadButton::Start: // Don't consume Start - let game handler pause/unpause return false; } } return false; // Let other handlers process unhandled input }
virtual bool HandleAxisInput(int gamepadId, GamepadAxis axis, float value, float delta) = 0

Processes analog axis input events from gamepads. Handles stick movements, trigger presses, and other analog inputs.

gamepadId: Unique identifier of the gamepad that generated the event
axis: Which axis changed (see GamepadAxis enum)
value: Current value of the axis (typically -1.0 to 1.0 for sticks, 0.0 to 1.0 for triggers)
delta: Change in axis value since the previous frame
Returns: true if the input was handled and should not be passed to lower priority handlers, false to continue processing
// Example axis handling for camera control bool CameraInputHandler::HandleAxisInput(int gamepadId, GamepadAxis axis, float value, float delta) { if (!IsActive()) return false; switch (axis) { case GamepadAxis::RightX: // Horizontal camera rotation (yaw) if (std::abs(value) > 0.1f) { // Simple deadzone check camera->adjustYaw(value * mouseSensitivity * deltaTime); return true; // Consume input } break; case GamepadAxis::RightY: // Vertical camera rotation (pitch) if (std::abs(value) > 0.1f) { camera->adjustPitch(-value * mouseSensitivity * deltaTime); // Invert Y return true; // Consume input } break; case GamepadAxis::LeftTrigger: // Zoom in while holding left trigger if (value > 0.1f) { camera->setZoom(1.0f - (value * 0.5f)); // Reduce FOV for zoom return true; // Consume input } break; } return false; // Let other handlers process unhandled axes }

Virtual Methods (Optional Override)

virtual void OnUpdate(double deltaTime) {}

Called at the beginning of each frame update cycle. Use this for time-based processing, state updates, or cleanup operations.

deltaTime: Time elapsed since the previous frame (in seconds)
// Example update implementation for animation void UIInputHandler::OnUpdate(double deltaTime) { // Update button highlight animations if (highlightTimer > 0.0f) { highlightTimer -= deltaTime; if (highlightTimer <= 0.0f) { ui->clearButtonHighlight(); } } // Update controller vibration duration if (vibrationDuration > 0.0f) { vibrationDuration -= deltaTime; if (vibrationDuration <= 0.0f) { gamepad->StopVibration(); } } }
virtual void OnGamepadConnected(int gamepadId) {}

Called when a gamepad is connected to the system. Use this to initialize gamepad-specific state or notify the user.

gamepadId: Unique identifier of the newly connected gamepad
// Example connection handling void PlayerInputHandler::OnGamepadConnected(int gamepadId) { std::cout << "Player controller " << gamepadId << " connected!" << std::endl; // Assign to a player slot int playerSlot = findAvailablePlayerSlot(); if (playerSlot != -1) { assignGamepadToPlayer(gamepadId, playerSlot); ui->showPlayerJoinedMessage(playerSlot); // Welcome vibration feedback auto* gamepad = gamepadManager->GetGamepad(gamepadId); if (gamepad && gamepad->SupportsVibration()) { gamepad->SetVibration(0.5f, 0.5f); vibrationDuration = 0.2f; // Stop after 200ms } } }
virtual void OnGamepadDisconnected(int gamepadId) {}

Called when a gamepad is disconnected from the system. Use this to handle player dropouts or pause the game.

gamepadId: Unique identifier of the disconnected gamepad
// Example disconnection handling void PlayerInputHandler::OnGamepadDisconnected(int gamepadId) { std::cout << "Player controller " << gamepadId << " disconnected!" << std::endl; // Find which player lost their controller int playerSlot = findPlayerByGamepadId(gamepadId); if (playerSlot != -1) { // Pause the game and show reconnection prompt game->pauseForReconnection(playerSlot); ui->showControllerDisconnectedMessage(playerSlot); // Mark player as awaiting reconnection players[playerSlot].awaitingReconnection = true; players[playerSlot].gamepadId = -1; } }
virtual int GetPriority() const { return 0; }

Returns the priority value for this handler. Higher values have higher priority and are processed first.

Returns: Priority value (typically 0-100, but can be any integer)
Common Priority Conventions:
  • 100-200: Critical UI elements (modal dialogs, system menus)
  • 50-99: Normal UI elements (HUD, game menus)
  • 10-49: Gameplay systems (player input, camera control)
  • 0-9: Background systems (analytics, debugging)
// Priority examples class CriticalUIHandler : public IGamepadInputHandler { public: int GetPriority() const override { return 150; } // Highest priority }; class GameMenuHandler : public IGamepadInputHandler { public: int GetPriority() const override { return 75; } // High priority }; class PlayerInputHandler : public IGamepadInputHandler { public: int GetPriority() const override { return 25; } // Medium priority }; class AnalyticsHandler : public IGamepadInputHandler { public: int GetPriority() const override { return 5; } // Low priority };
virtual bool IsActive() const { return true; }

Returns whether this handler is currently active and should process input. Inactive handlers are skipped entirely.

Returns: true if the handler should process input, false to skip it
// Example active state management class MenuInputHandler : public IGamepadInputHandler { bool menuVisible = false; public: bool IsActive() const override { return menuVisible && !game->isPaused(); } void showMenu() { menuVisible = true; } void hideMenu() { menuVisible = false; } };
virtual const char* GetName() const { return "Unknown Handler"; }

Returns a human-readable name for this handler. Used for debugging and logging purposes.

Returns: Null-terminated string containing the handler name
// Example name implementation class PlayerMovementHandler : public IGamepadInputHandler { public: const char* GetName() const override { return "Player Movement Handler"; } };

GamepadInputHandler Base Class

class GamepadInputHandler : public IGamepadInputHandler

Purpose: Concrete base implementation that provides common functionality and default implementations for the IGamepadInputHandler interface. Most custom handlers should inherit from this class rather than implementing IGamepadInputHandler directly.

Constructor

GamepadInputHandler(const std::string& name = "BaseHandler", int priority = 0, bool active = true)

Constructs a GamepadInputHandler with the specified configuration.

name: Human-readable name for debugging (default: "BaseHandler")
priority: Processing priority (default: 0)
active: Whether the handler starts in active state (default: true)
// Constructor examples auto menuHandler = std::make_shared ("MainMenu", 75, true); auto gameHandler = std::make_shared ("PlayerControl", 25, false); // Start inactive auto debugHandler = std::make_shared("Debug Console", 5);

Default Implementation

bool HandleButtonInput(...) override { return false; }
bool HandleAxisInput(...) override { return false; }

Base implementations that do nothing and allow all input to pass through. Override these in derived classes to provide actual functionality.

Important: The base class implementations return false (don't consume input). Always call the base implementation or return false for inputs you don't handle to maintain the input chain.

Accessor Methods

bool IsActive() const override
void SetActive(bool active)

Get or set the active state of the handler. Inactive handlers are completely skipped during input processing.

// Dynamic activation control menuHandler->SetActive(false); // Disable menu input gameHandler->SetActive(true); // Enable game input if (pauseMenuOpen) { menuHandler->SetActive(true); // Re-enable menu input gameHandler->SetActive(false); // Disable game input }
int GetPriority() const override
void SetPriority(int priority)

Get or set the priority of the handler. Changes take effect on the next frame when the handler stack is resorted.

// Dynamic priority adjustment if (criticalDialogOpen) { dialogHandler->SetPriority(200); // Boost to highest priority } else { dialogHandler->SetPriority(50); // Normal UI priority }
const char* GetName() const override
void SetName(const std::string& name)

Get or set the human-readable name of the handler.

// Dynamic naming for debugging handler->SetName("Player" + std::to_string(playerId) + "Input");

Usage Patterns

Simple Handler Implementation

// Basic handler that inherits from GamepadInputHandler class SimplePlayerHandler : public GamepadInputHandler { public: SimplePlayerHandler() : GamepadInputHandler("SimplePlayer", 20) {} bool HandleButtonInput(int gamepadId, GamepadButton button, bool pressed, bool justPressed, bool justReleased) override { if (!IsActive()) return false; if (justPressed) { switch (button) { case GamepadButton::A: player->jump(); return true; // Consume jump input case GamepadButton::B: player->attack(); return true; // Consume attack input } } return false; // Let other handlers process unhandled buttons } bool HandleAxisInput(int gamepadId, GamepadAxis axis, float value, float delta) override { if (!IsActive()) return false; switch (axis) { case GamepadAxis::LeftX: case GamepadAxis::LeftY: // Movement input - always consume if active player->setMovementInput( gamepad->GetLeftStickX(), gamepad->GetLeftStickY() ); return true; } return false; // Let other handlers process unhandled axes } };

State-Aware Handler

// Handler that changes behavior based on game state class AdaptiveUIHandler : public GamepadInputHandler { GameState currentState = GameState::Menu; public: AdaptiveUIHandler() : GamepadInputHandler("AdaptiveUI", 80) {} void setGameState(GameState state) { currentState = state; // Adjust priority based on state switch (state) { case GameState::Menu: SetPriority(100); // High priority in menus break; case GameState::InGame: SetPriority(60); // Lower priority during gameplay break; case GameState::Paused: SetPriority(90); // High priority when paused break; } } bool HandleButtonInput(int gamepadId, GamepadButton button, bool pressed, bool justPressed, bool justReleased) override { if (!IsActive() || !justPressed) return false; switch (currentState) { case GameState::Menu: return handleMenuInput(button); case GameState::InGame: return handleGameUIInput(button); case GameState::Paused: return handlePauseInput(button); } return false; } private: bool handleMenuInput(GamepadButton button) { switch (button) { case GamepadButton::A: menu->selectItem(); return true; case GamepadButton::B: menu->goBack(); return true; case GamepadButton::DPadUp: menu->navigateUp(); return true; case GamepadButton::DPadDown: menu->navigateDown(); return true; } return false; } bool handleGameUIInput(GamepadButton button) { switch (button) { case GamepadButton::Start: game->showPauseMenu(); setGameState(GameState::Paused); return true; case GamepadButton::Back: hud->toggleInventory(); return true; } return false; } bool handlePauseInput(GamepadButton button) { switch (button) { case GamepadButton::Start: game->resumeGame(); setGameState(GameState::InGame); return true; case GamepadButton::A: pauseMenu->selectItem(); return true; } return false; } };

Integration with GamepadManager

// Complete integration example class Game { GamepadManager gamepadManager_; std::shared_ptr uiHandler_; std::shared_ptr playerHandler_; std::shared_ptr debugHandler_; public: void initialize() { // Initialize gamepad manager gamepadManager_.Initialize(); // Create input handlers uiHandler_ = std::make_shared ("GameUI", 75); playerHandler_ = std::make_shared ("PlayerControl", 25); debugHandler_ = std::make_shared ("Debug", 5); // Add handlers to manager (they will be sorted by priority) gamepadManager_.AddInputHandler(uiHandler_); gamepadManager_.AddInputHandler(playerHandler_); gamepadManager_.AddInputHandler(debugHandler_); // Set up global callbacks for system-level events gamepadManager_.SetConnectedCallback([this](int gamepadId) { std::cout << "System: Gamepad " << gamepadId << " connected" << std::endl; }); gamepadManager_.SetDisconnectedCallback([this](int gamepadId) { std::cout << "System: Gamepad " << gamepadId << " disconnected" << std::endl; }); } void update(double deltaTime) { // Update gamepad manager - this will: // 1. Update all connected gamepads // 2. Process input through all active handlers in priority order // 3. Call OnUpdate on all active handlers gamepadManager_.Update(deltaTime); } void showMenu() { uiHandler_->SetActive(true); // Enable UI input playerHandler_->SetActive(false); // Disable player input } void startGame() { uiHandler_->SetActive(false); // Disable UI input playerHandler_->SetActive(true); // Enable player input } void showDebugConsole() { debugHandler_->SetPriority(150); // Boost debug priority to highest } void hideDebugConsole() { debugHandler_->SetPriority(5); // Reset debug priority to lowest } };

Best Practices

Input Consumption Strategy

  • Be selective: Only consume input that your handler actually processes
  • Common inputs: Consider not consuming common inputs like Start/Pause that should be available system-wide
  • Context awareness: Consume more aggressively when your system has focus (e.g., modal dialogs)
  • Debugging: Log input consumption decisions during development to debug input flow issues

Priority Management

  • Use ranges: Group related handlers in priority ranges (UI: 50-100, Game: 10-49, etc.)
  • Leave gaps: Use priorities like 10, 20, 30 to leave room for future handlers
  • Dynamic adjustment: Change priorities based on game state when needed
  • Document priorities: Maintain a list of priority conventions for your project

Performance Considerations

  • Early returns: Return early in HandleButtonInput/HandleAxisInput when IsActive() is false
  • Minimize work: Keep input processing lightweight - defer heavy operations to Update()
  • Avoid allocations: Don't allocate memory in input handling methods
  • Cache lookups: Cache expensive lookups rather than repeating them every frame

Architecture Guidelines

  • Single responsibility: Each handler should have a clear, focused purpose
  • State management: Use IsActive() to enable/disable handlers rather than removing them
  • Lifecycle: Store handlers as shared_ptr to manage lifetime properly
  • Testing: Design handlers to be testable by injecting dependencies rather than accessing globals

Common Patterns

  • UI Overlay: High priority handler for modal dialogs that consumes all input
  • Menu System: Medium priority handler that consumes navigation but passes through system inputs
  • Player Control: Lower priority handler that processes gameplay input
  • Debug/Analytics: Lowest priority handler that logs input without consuming it

Integration Tips

  • Handler lifetime: Create handlers during initialization and keep them alive throughout the game
  • State synchronization: Update handler active states when game states change
  • Error handling: Wrap input processing in try-catch blocks for robustness
  • Debugging: Use GetName() and GetPriority() for debug output and logging
